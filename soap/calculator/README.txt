
1)Сервер
1.1) @WebService - аннотация, которая готоврит, что данный класс может быть использован как движок для сервера
1.2) @WebMethod - по дефолту все методи в класс, помеченным аннотацией @WebService не видны слиенту(закрты для его использования)
Нужно явно добавить эту аннотацию для метода
1.3) Endpoint.publish("url", class) - запускает простейший веб сервер и в нео деплоит наш веб сервис


2) Проверка
Что бы проверить, что soap web service запущен - нужно просто в браузере написть url+?wsdl
http://127.0.0.1:8083/calculator?wsdl

3) SoapUI - инструмент для проверки веб сервисов
В верхнем углу нажимаешь SOAP
name - просто имя
wsdl: http://127.0.0.1:8083/calculator?wsdl
(только нужно запустить сервер)

И потом пооткрывать + и дойти до Request и вставить нужные значения


4) Клиент на Java
4.1) вызываешь cmd
создадим директорию на диске С -> qqq
переходим в нее
cd C:\qqq
4.2) теперь запускаем команду
wsimport http://127.0.0.1:8083/calculator?wsdl -keep
В результате в этой же директории появится папка (в нашем случае com)
4.3) Внутри этой папки нужно удалить все .class файлы
4.4) Создаем новый проект
my-client
и копируем папку com из шага 4.2) в src/main/java
должно получится src/main/java
